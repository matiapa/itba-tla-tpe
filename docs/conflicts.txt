Si queremos soportar por ej. "write 2+2" hay que tener las reglas:
    write: WRITE expression  [1]
    expression: SYMBOL_NAME  [2] (Validando tipo 'number')
Pero si se hiciera "write text_var" entonces se reduciría usando [2] y daría error.

Se podría agregar una regla para las variables texto:
    write: WRITE SYMBOL_NAME [3]
Pero se produciría un conflicto RR entre [1] y [3] al tenerse un SYMBOL_NAME.

Habría que saber antes de aplicar una reducción si SYMBOL_NAME corresponde a una variable
de tipo numérico o texto. Como esto no es posible (creo) lo solucioné no permitiendo realizar
un write de una expresión directamente, solo se puede imprimir texto o una variable.

number n1 = 2
number n2 = 2 * n1
